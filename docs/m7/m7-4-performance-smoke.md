# M7-4 性能冒烟记录（执行耗时 P95）

- 里程碑: Plan / M7-4
- 目标: 固定基准样例执行耗时可接受（建议 P95 < 1.5s）。
- 测试构建: debug
- 应用版本: 1.0
- 设备型号: PLQ110 (OnePlus Ace 6)
- Android 版本: 16
- 测试日期: 2026-03-28
- 测试人: Guyuepp

## 指标定义

1. 本文统计的是应用内端到端执行耗时（Elapsed / durationMs）。
2. 指标包含请求调度、Python 调用与结果回传开销。
3. 该指标不是网络延迟，也不等于纯 Python 函数自身运行时间。

## 测试输入

- 固定基准样例（本次实测）:

```python
sum(i * i for i in range(4000))
```

- 说明:
  1. 统一输入样例，减少结果偏差。
  2. 同一设备上连续执行 20 次。
  3. 本样例为轻载表达式，结果用于冒烟基线，不代表复杂脚本极限性能。

## 执行步骤

1. 清后台，保持设备电量和温度稳定。
2. 在同一外部 App 中选中文本，触发 Run as Python。
3. 每次记录结果页 `Elapsed: xxx ms` 数值。
4. 连续记录 20 次，必要时记录异常（超时、取消、崩溃）。

## 统计方法

1. 将 20 个 durationMs 升序排序。
2. P95 位置可取 `ceil(20 * 0.95) = 19`。
3. 排序后第 19 个值作为 P95。

## 统计结果

- 样本量: 20
- 最小值(min): 6 ms
- 最大值(max): 24 ms
- 平均值(avg): 14.4 ms
- P95: 21 ms

## 验收结论

- 阈值: P95 < 1500 ms
- 本次结论: Pass
- 风险与说明:
  1. 本次样本均为 success，未覆盖超时/异常场景下的性能波动。
  2. 数据来自单设备（PLQ110）与单日测试，建议后续补充至少一台中低端机复测。
  3. 当前 P95 与阈值差距充足（21 ms vs 1500 ms），可视为通过但仍需发布前复核一次。
